1) Описание проблемы: При использовании метода contains класса StringUtils с пустой строкой или строкой, 
состоящей только из пробелов, ожидаемый результат должен быть False. Однако в некоторых случаях тесты, 
использующие параметры ("", "", False) и (" ", "", False), возвращают неверный результат,
указывая на True вместо ожидаемого False.
Ожидаемое поведение: Метод contains должен возвращать False при поиске символа в пустой строке или строке,
состоящей только из пробелов.
Фактическое поведение: В некоторых случаях тесты возвращают результат True вместо ожидаемого False 
для пустой строки и строки, состоящей только из пробелов.
